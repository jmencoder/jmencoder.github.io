Git FTP para desplegar tus proyectos a producción

Muchas veces nos encontramos con el reto de buscar la mejor manera de desplegar nuestros proyectos algunos optan por la eficiencia otros por la facilidad o como me ocurrió a mi por que simplemente no conocía otra forma para realizar esta tarea.

Si utilizas Git y aún sigues cargando tus archivos por FTP entonces dale una oportunidad a esta excelente herramienta.

Que es Git FTP ?

En pocas palabras es un cliente FTP la diferencia es que esta herramienta puede ahorrarte algo de tiempo y ancho de banda al cargar solo los archivos que cambiaron desde la última carga.

Realiza un seguimiento de los archivos cargados almacenando la identificación de confirmación en un archivo de registro en el servidor. Utiliza Git para determinar qué archivos locales han cambiado.

Puede implementar fácilmente otra rama o volver al historial de Git para cargar una versión anterior.

Asi es amiga(a), tal y como lo imaginas tienes todas las cualidades que Git te ofrece y también tienes una potente herramienta de FTP.

Como lo instalamos ?

Descargamos el repositorio para aplicarlo a nuestro equipo, para hacerlo simpmenete nos dirigimos al siguiente repositorio en Github

https://github.com/git-ftp/git-ftp

Para descargarlo hacemos click en el botón CODE y luego en la opción Download Zip

Cuando ya lo tenemos descargado copiamos el git-ftp-master.zip a la carpeta bin del directorio donde se instalo Git, si no cambiaste la ruta por defecto donde se instala Git entonces será la siguiente

C:\Program Files\Git\bin

Una vez tenemos copido el archivo git-ftp-master.zip a la carpeta bin, descomprimimos el archivo y renombramos la carpeta a git-ftp.

Listo! con estos sencillos pasos habremos habilitado Git-FTP

Para comprobarlo abrimos la terminal que se instala con Git llamada Git Bash y ejecutamos el siguiente comando

git ftp -v

Y el resultado debe ser el siguiente

Posible error

Si por alguna razón no llegamos a ver el mensaje que se muestra en la imagen anterior y se muestra vació o nos da un mensaje como este

" is not a git command "

La solución es simple

Para solucionar esto debemos copiar la carpeta que renombramos git-ftp y pegar la copia dentro del siguiente directorio

C:\Program Files\Git\mingw64\libexec\git-core\

Si hicimos todo correctamente y volvemos a ejecutar el comando git ftp -v deberíamos obtener el resultado que mencionamos anteriormente.
